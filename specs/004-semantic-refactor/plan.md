# Implementation Plan: Semantic System Refactor

**Branch**: `004-semantic-refactor` | **Date**: 2025-01-29 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-semantic-refactor/spec.md`
**Reference**: [ADR-SEMANTIC-TYPE-STATUS-ONTOLOGY.md](../../docs/ADR-SEMANTIC-TYPE-STATUS-ONTOLOGY.md)

## Summary

Simplify the EntityState structure by removing unused abstractions (EntityClass, EntityRole, EntityStatus, NodeProperties, Position) and clarifying framework vs domain responsibilities. The refactor promotes ID to top-level, replaces ObjectRef with proper StorageReference, and removes hardcoded status generation.

## Technical Context

**Language/Version**: Go 1.25+
**Primary Dependencies**: NATS JetStream (KV buckets), GraphQL (gqlgen)
**Storage**: NATS KV (ENTITY_STATES bucket)
**Testing**: Go test with `-race` flag, table-driven tests
**Target Platform**: Linux server (edge devices, cloud)
**Project Type**: Single Go module with internal packages
**Performance Goals**: Existing performance maintained (no regression)
**Constraints**: Greenfield - no backward compatibility required
**Scale/Scope**: ~50 files affected across graph/, message/, processor/, gateway/ packages
