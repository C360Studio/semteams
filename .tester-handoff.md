All three test suites follow the established pattern from `processor/rule/rule_integration_test.go`:

```go
// Shared TestMain with testcontainers
func TestMain(m *testing.M) {
    testClient, _ := natsclient.NewSharedTestClient(
        natsclient.WithJetStream(),
        natsclient.WithKV(), // For loop tests
        natsclient.WithKVBuckets("AGENT_LOOPS", "AGENT_TRAJECTORIES"),
        natsclient.WithStreams(streams...),
    )
    sharedTestClient = testClient
    exitCode := m.Run()
    testClient.Terminate()
}
```

**Key patterns used**:
- `//go:build integration` tag on all files
- `require.Eventually()` for async NATS message assertions
- Unique subject names per test to prevent collisions
- Mutex-protected shared state for goroutine safety
- Real NATS JetStream via testcontainers (no mocks)

## Integration Requirements

### NATS Infrastructure

**Streams**:
- `AGENT` stream with subjects: `agent.>`, `tool.>`

**KV Buckets** (loop tests only):
- `AGENT_LOOPS` - stores LoopEntity state
- `AGENT_TRAJECTORIES` - stores completed Trajectory

**Subjects**:
- `agent.task.*` - task submissions
- `agent.request.*` - model requests
- `agent.response.*` - model responses
- `tool.execute.*` - tool execution requests
- `tool.result.*` - tool results
- `agent.complete.*` - completion events
